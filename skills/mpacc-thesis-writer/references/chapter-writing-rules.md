# Chapter Writing Rules

Write chapters as connected work packages. Do not draft isolated essays.

## Chapter 1: Introduction

Include background, significance, literature overview, research content, methods, technical route, and innovation.

- Background should move from the specific case problem outward as needed. Do not spend pages proving a broad field is important while the case problem remains unclear.
- State the reality problem, the key associated factor(s), the research question, and the provisional answer/main claim.
- Significance should separate theoretical and practical value.
- Methods must match evidence: case analysis, comparative analysis, event study, AHP, fsQCA, machine learning, social network analysis, interview, questionnaire, or document analysis.
- Innovation should be modest: new case, new data, new framework combination, or application of a new policy/standard.
- Innovation must not depend on invented concepts, data stretching, or a method label alone.

Good: "The study uses process documents, financial indicators, and interview evidence to evaluate budget compilation pain points."

Weak: "The study uses various scientific methods to conduct comprehensive analysis."

## Chapter 2: Literature, Concepts, And Theory

Define concepts first, synthesize literature by themes, then select theories. Theory must be used later.

Avoid listing one paper per paragraph. End with a review that identifies a usable gap.

## Chapter 3: Case/Current Situation

Describe the case only as needed for later diagnosis:

- organization and business model;
- case design reason and evidence boundary;
- current process or accounting treatment;
- data sources and time span;
- current indicators, risk points, or governance arrangements.

Use tables for process, indicators, and data inventory.

## Chapter 4: Problem And Cause Analysis

Problem statements should be evidence-based symptoms. Causes should explain why symptoms occur.

Good: "Budget deviation is concentrated in medicine procurement and department-level projects; causes include single historical-average forecasting and weak business-data linkage."

Weak: "The company should improve awareness and strengthen management."

Avoid the working-report pattern of many parallel problems, many causes, and many measures. Select the core problem and one or two main associated factors, then prove them deeply.

For non-optimization theses, rename the chapter function instead of forcing "problems and causes":

- event/spillover: direct effect, spillover effect, and mechanism;
- model/accounting treatment: model construction, recognition/measurement path, or accounting treatment design;
- configuration: condition selection, calibration, necessity, and configuration analysis;
- staged path: path mechanism by stage;
- intelligent model: indicator construction, model training/comparison, and case application.

## Chapter 5: Scheme, Model, Or Effect Evaluation

Choose one role:

- optimization scheme with safeguards;
- accounting/model construction with case application;
- mechanism/effect evaluation with indicators and robustness/contrast.

Connect every design item to a problem from Chapter 4.

For tool applications, include industry-specific indicators, threshold basis, implementation owner, data path, trigger action, and expected effect.

If Chapter 4 is a method/model chapter, Chapter 5 should show application, validation, interpretation, and management implications.

## Chapter 6: Conclusion

Answer the research question, summarize findings, state limitations, and suggest future research. Do not introduce new evidence.

## Drafting Discipline

- Write with placeholders for missing facts.
- Keep citations tied to claims, not decorative.
- Prefer "evidence -> interpretation -> implication" paragraphs.
- Refresh the project snapshot after finishing each chapter.
