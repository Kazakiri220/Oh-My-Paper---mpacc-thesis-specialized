## S01 A 市财政衔接资金绩效评价体系优化研究

- Research type: public-sector performance evaluation optimization.
- Case/evidence base: A city fiscal linkage-fund performance evaluation project, budget-performance information system, process evidence, indicator system, interviews, and project-practice experience.
- Theory/framework: principal-agent, new public management, public finance, stakeholder, performance management; AHP and big-data/text classification as tools.
- Structure: introduction and literature; concepts/theory/tools; current evaluation system; mechanism optimization and indicator-library construction; application comparison and safeguards; conclusion.
- Diagnosis logic: third-party evaluators lack project experience; evaluation results lack data support; indicators and weights are unreasonable; causes include weak selection/supervision mechanisms and incomplete performance knowledge base.
- Solution logic: optimize evaluation mechanism, build data-driven indicator library, reconstruct weighted indicators, reapply to the project, then add safeguards for awareness, platform, talent, responsibility, and feedback.
- Reusable rule: an optimization thesis is strongest when the proposed framework is reapplied to the original case and compared with the old result.
- Do not reuse: city-specific scores, project indicators, interview details, or fund data.

## Cross-Case Rules Added After Full Reading

1. Match the official thesis form and analytical pattern before outlining. Analytical patterns include optimization, mechanism/effect, model/accounting treatment, event/spillover, configuration, or intelligent-technology application.
2. Require a method-data fit table for any thesis using AHP, fuzzy AHP, fsQCA, event study, text analysis, machine learning, social network analysis, or carbon/accounting model construction.
3. For digital/AI topics, force the technology to serve an MPAcc decision: budgeting, risk identification, cost warning, audit evidence, performance evaluation, accounting treatment, or disclosure.
4. For case optimization, require old state -> diagnosis -> cause -> design -> application or expected effect -> safeguard.
5. For mechanism theses, require path variables and effect indicators; avoid direct "background causes performance" leaps.
6. For accounting-treatment theses, require economic substance, standard mapping, recognition timing, measurement attribute, presentation/disclosure, and case effect.
7. For model theses, require input data, indicator construction, model choice, validation/application, interpretability, and management action.
8. For empirical/configuration/event theses, require sample/event boundary, variable or condition definition, method rationale, result interpretation, and limitations.
