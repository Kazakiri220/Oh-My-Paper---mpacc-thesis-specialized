# Method Data Fit Guide

Read this before promising a method-heavy MPAcc thesis. Full-text reading showed that excellent theses make the method serve a concrete accounting, audit, finance, cost, risk, budget, disclosure, or performance decision.

Core rule: choose the method after the real problem, research question, and evidence boundary are clear. A fashionable method does not compensate for a weak problem. If the method does not change diagnosis, solution design, accounting treatment, evaluation, or decision making, remove it.

## Fit Table

| Method | Minimum Data Needed | Output | Common Failure |
|---|---|---|---|
| AHP / fuzzy AHP | indicator hierarchy, expert scores, scoring basis, case evidence | weighted indicators, evaluation result, improvement priorities | weights appear without expert/source explanation or are assigned by the author |
| Event study | clear event date, market data, estimation window, comparison logic | market reaction and interpretation | event date is vague or confounded |
| fsQCA | comparable cases, condition definitions, calibration anchors, outcome measure | high/non-high outcome configurations | conditions are chosen only because data is available |
| Machine learning | sample size, features, labels, training/testing split or validation plan | prediction/warning model and interpretable risk drivers | model accuracy is reported without management meaning or enough data |
| Text analysis / MD&A | text source, cleaning method, dictionary/model, text indicators | incremental non-financial indicator | text tone is added without proving added value |
| Social network analysis | entity/person/transaction relationship data, node/edge definitions | suspicious related-party clues and paths | network diagram replaces audit reasoning |
| Carbon/cost accounting model | boundary, cost object, emission/cost factors, stage data | cost list, calculation model, stage control point | calculation boundary is unclear |
| Accounting treatment model | economic substance, standards/policy mapping, recognition and measurement evidence | recognition timing, measurement attribute, presentation/disclosure | policy summary has no case application |
| Resource/path case | stage boundary, drivers, resources, capabilities, effect indicators | staged mechanism and effect evaluation | company history is mistaken for analysis |
| KRI / warning dashboard / risk map | real risk event or visible standard indicator, industry threshold basis, owner, data path, trigger action | management warning tool and control response | generic ratios are listed and falsely treated as prediction |

## Method Acceptance Test

Before drafting, answer:

- What decision does this method support?
- What exact data/material does it need?
- Which chapter supplies the data?
- What result will it produce?
- How will the result change diagnosis, recommendation, accounting treatment, or evaluation?

Good: "Use fuzzy AHP because tracking-audit findings can be converted into staged internal-control indicators and expert scores can produce a prioritized risk result."

Weak: "Use machine learning because it is innovative."

## Method Red Lines

- Do not start topic selection from a preferred method such as AHP, fsQCA, machine learning, text mining, or DID.
- Do not use AHP or entropy weights without a defensible scoring source, survey/expert basis, or objective data basis.
- Do not call a management warning tool a statistical prediction model unless it has training data, validation, and out-of-sample logic.
- Do not use big-sample empirical work unless it has clear practice guidance and still serves the MPAcc problem-solving goal.
- Do not add LLM-assisted coding or text analysis without source texts, coding rules, human review, and validation.
- Do not let a model replace accounting reasoning. Explain what the result means for recognition, measurement, disclosure, control, risk response, budget, cost, audit evidence, or performance decision.
